Submit Distributed Computing Process

Using this file you can submit your process to the distributed computing mechanism. After submitting you will receive a process code in order to activate.

  • Try keeping the name short and Simple. The name should generally hint on the process destiny.

  • The trim function will divide your global task into mini-tasks. Trim function receives the main parameter and the machine count you would like to work on. Make sure you keep the function definition as it is.

  • The map function runs on each of the distributed computing nodes. In other words these are the little tasks. Make sure you keep the function definition as it is.

  • The reduce function is the arrangement of answers function. this function receives answers from all nodes and work them into one final answer. Make sure you keep the function definition as it is.
